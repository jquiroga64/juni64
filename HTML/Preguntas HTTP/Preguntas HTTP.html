Preguntas sobre HTTP

1) Sobre qué protocolo binario está montado el protocolo HTTP?

HTTP se basa en TCP (Transmission Control Protocol) para la transmisión de datos. TCP es un protocolo de comunicación orientado a conexión que asegura la entrega de datos.

2) ¿Cuáles son los clientes HTTP y los servidores HTTP más utilizados?

Clientes HTTP:

  • Navegadores web (Chrome, Firefox, Safari, Edge).
  • Herramientas de línea de comandos (cURL, Wget).
  • Aplicaciones móviles y de escritorio.

Servidores HTTP:

  • Apache HTTP Server.
  • Nginx.
  • Microsoft IIS.
  • LiteSpeed.

3) ¿Qué verbos admite un comando en el requerimiento HTTP?

  • GET: Solicitar datos del servidor.
  • POST: Enviar datos al servidor para crear o actualizar un recurso.
  • PUT: Actualizar un recurso existente.
  • DELETE: Eliminar un recurso.
  • HEAD: Solicitar los headers de una respuesta sin el cuerpo.
  • OPTIONS: Obtener información sobre las opciones de comunicación disponibles.

4) ¿Qué contenido lleva el body de un requerimiento HTTP?

El body de un requerimiento HTTP puede contener datos que el cliente envía al servidor, como:

  • Datos de formularios (en formato application/x-www-form-urlencoded o multipart/form-data).
  • JSON (en formato application/json).
  • XML (en formato application/xml).
  • Archivos binarios.

5) ¿Qué diferencia existe entre un URL y un URI?

URI (Uniform Resource Identifier): Es un identificador genérico que puede ser una URL o un URN (Uniform Resource Name).

URL (Uniform Resource Locator): Es un tipo específico de URI que proporciona la dirección de un recurso y cómo acceder a él (por ejemplo, http://example.com).

6) ¿Cómo se almacena información relacionada con las respuestas HTTP?

La información de las respuestas HTTP se almacena en:

  • Headers de la respuesta: Incluyen metadatos sobre la respuesta (tipo de contenido, estado, cookies, etc.).
  • Cuerpo de la respuesta: Contiene el contenido real que se envía al cliente (HTML, JSON, imágenes, etc.).

7) ¿Qué significa Virtual Hosting?

El Virtual Hosting permite que un solo servidor web aloje múltiples dominios o sitios web. Esto se logra mediante la configuración de diferentes nombres de dominio y/o direcciones IP en un mismo servidor.

8) ¿Por qué el HTTP se considera STATELESS?

HTTP se considera stateless porque cada solicitud del cliente al servidor es independiente y no se mantiene información sobre las solicitudes anteriores. El servidor no guarda estado de la sesión del cliente, lo que simplifica el diseño y mejora la escalabilidad.

9) ¿Qué nueva versión de HTTP se está usando para mejorar la velocidad de la WEB?

HTTP/3 es la nueva versión que utiliza QUIC (Quick UDP Internet Connections) en lugar de TCP, lo que mejora la velocidad y la eficiencia en la transmisión de datos, especialmente en redes inestables.

10) ¿Cómo va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

HTTP/3, al utilizar QUIC, reduce la latencia y mejora la resistencia a la pérdida de paquetes. Esto permite que las aplicaciones web mantengan un rendimiento más constante y confiable, incluso en conexiones inestables o con alta latencia, al permitir múltiples flujos de datos dentro de una misma conexión.
